About

landialler enables several computers on a LAN (Local Area Network) to share control of a dial up connection to the Internet. To see if landialler would be useful to you, answer these questions:

  1. Do you have several computers at home, connected together in a small network?

  2. Do you connect to the Internet intermittently (rather than being permanently connected)?

  3. Can you configure your network so that all your computers can access the Internet at once? This is the trickiest bit, but is well worth the effort. There are lots of instructions on how to set it up with Linux.

If you said no to any of the above questions then landialler is probably not going to be of much use to you.

If you said yes (or nearly yes) to all of them, then landialler will enable any of the computers on your home network to control your Internet connection.

landialler comes in two parts:

  • The client
    The client is a small graphical application that is run on a computer without a modem.

  • The server
    The server is a daemon program (i.e. it is always running) that sits on the computer that has the modem, waiting for a client to ask it to connect to the Internet. Currently, only POSIX compliant operating systems (i.e. Unix) are supported. If there is demand for Windows support then it will be added in due course.

The client and server are both written in Python and communicate with each other using XML-RPC.

Screenshots

After "Yes" has been clicked on the first dialog the client sends an XML-RPC message to the server, instructing the modem to connect to the Internet. The server actually runs an operating system dependent script, so landialler can be easily configured to use different operating systems and different sort of dial up techniques.

Once the script has started and the server is dialling up, the client continually asks the server for an progress update. If the connection is made inside the (configurable) time limit then the second dialog pops up. To close the connection just click OK (yep, you're right - that button should say "Disconnect", but I need to learn some Tk first).

Finally, if the connection timed out you get the option to try again. You may want to tweak the time out value, which is easy because it's stored in the landialler configuration file. The default time out is two minutes, as that's what the author finds works rather well on his modem, with his ISP.
