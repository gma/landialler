    It's taken longer than I'd hoped, but a new Python 2.3 compatible version of LANdialler is now available from CVS (scroll down to the "Anonymous CVS Access" section).   • Development back underway (18 January 2004)
    I've made very few changes to LANdialler for a long time now, primarily because it just worked. However, it no longer works out of the box with the later versions of Python due to the absorption of the XML-RPC module into the Python standard library. This is the first thing that will be fixed. Note: The instructions below refer to running LANdialler with Python 2.1. If you want to use a later version of Python please wait a week or two until the new version is ready.

    I'm also intending to port the Unix GUI to GTK2, a change which is long over due. For a full list of what's coming see the TODO list.

About

[ more screenshots ]

LANdialler enables several computers on a LAN (Local Area Network) to share control of a dial up connection to the Internet.

If you have a small home or office network (a LAN) that connects to the Internet via a dial up link on a Linux computer, then LANdialler could be of real use to you. If you haven't got a Linux router yet find out how to set one up. The LANdialler client software should run on most operating systems, including Windows and Linux.

LANdialler is released under the terms of the GNU General Public License.

How does it work?

LANdialler consists of two programs:

  • The client is run on all desktop computers that require Internet access. It talks to the server, requesting Internet connectivity for as long as the user requires it. The interface is very simple, which is officially a "good thing".

  • The server runs on the Linux router and manages the state of the dial up connection. It keeps track of the number of people who have asked to go online and keeps the Internet connection up until the last user disconnects.

Because the dial up connection is handled by your Linux router your desktop computers can remain safely protected from prying eyes by your router's firewall (assuming you have configured it - if not why not?).

The client and server communicate with one another over the local network through the XML-RPC protocol.

Download

The latest versions of LANdialler are always available from the project files page. You should download the latest versions of all three packages. Installation instructions are included with the source code.

Requirements

Both the client and server are written in the Python programming language. Please make sure that you have a recent version (2.1 or later) installed. You can download it for free.

The client should run on any operating system with Python and Tk support - it has been tested on both Windows and Linux.

The server should run on any POSIX compliant operating system, but has only been tested on Debian GNU/Linux v2.2. You could use any of the BSD variants (e.g. FreeBSD, OpenBSD, NetBSD) or Solaris, etc. instead if you prefer (success reports would be very welcome). Just make sure you can configure it to route IP packets between your network and the Internet. There is lots of documentation available for Linux if you're not sure how to set it up.

Both the client and the server require PythonWare's xmlrpclib library.
